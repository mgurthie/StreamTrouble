Streaming Verification Guide

Use the symptom-driven runbooks to verify, isolate, document, and escalate live-streaming incidents.

Start Troubleshooting Open Operator Tools
Observe→Verify→Isolate→Escalate→Document

How to Use This Guide

1

Observe

Identify the reported symptom and affected stream.

2

Verify

Use the Station Website, Quickplay, ADit, mobile, and OTT tools.

3

Isolate

Determine whether the issue is viewer, platform, broadcast, or system-wide.

4

Escalate

Contact the correct team with dashboard evidence and exact times.

5

Document

Capture screenshots, ticket numbers, actions, and final resolution.

Operator Tools

Quickplay Dashboard

Verify channel health, ingest, manifest updates, playback preview, and active alarms.

  • Channel Healthy
  • Ingest Active
  • Manifest Updating
  • Playback Available
  • No Active Alarms

ADit Dashboard

Verify playlist progression, current events, break timing, and missed events.

  • Current Playlist
  • Current Event
  • Playlist Advancing
  • Commercial Timing
  • No Missed Events

Station Website

Confirm live playback, audio/video, program match, slate, freeze, or buffering.

Mobile & OTT

Compare the same stream on mobile, Roku, Fire TV, and Apple TV.

Confidence Monitor

Compare OTA output with the digital stream to separate broadcast and streaming failures.

Stream Down

When to use: Use this page when the live stream will not start, displays a black screen, or produces a playback error.

Symptoms

  • Black screen
  • Playback error
  • Stream unavailable

Recommended Check Order

  1. Confirm OTA source and routing.
  2. Verify Harmonic X2S input/output health.
  3. Check Quickplay ingest and manifest.
  4. For break issues, inspect ADit event timing.
  5. Check GAM after cue delivery is confirmed.
  6. Check CDN/player telemetry.

Evidence to Capture

  • Exact incident time and affected stream
  • ADit current event or break state
  • Quickplay health and active alarms
  • Harmonic alarms, input lock, output status
  • Platform comparison and screenshot
